### 技术栈与环境
- C++标准：优先使用 C++17
- 编译器：Windows平台使用 MinGW-w64 (GCC)
- IDE/编辑器：VS Code

### 基础格式规范
- 缩进：统一使用 **4个空格**
- 大括号：**所有左大括号必须单独换行**（Allman风格）
- 右大括号单独一行
- 运算符前后必须加空格
- 逗号后必须加空格
- 引用符号 `&` 两边必须加空格
- 关键字（if, for, while, switch等）后面必须加空格
- 代码开头**强制添加 using namespace std;**，全程使用标准命名空间
- main 函数末尾**强制写 return 0;**，不可省略

### 排版空行强制规则（补充注释分隔规则）
1. **无注释场景**：不同函数定义之间，空一行直接分隔
2. **带注释场景（注释归属下方函数）**：上一个函数结束 → 空一行 → 写函数注释 → 再空一行 → 下一个函数定义，以此区分注释归属
3. **函数内部代码保持紧凑**：if、for、while、switch 等流程语句，上下不额外空行
4. 仅逻辑差异极大的代码块可少量空行，其余保持紧凑书写

### 括号与空格终极规则（100%精准匹配）
#### 通用括号内规则
- **只有当括号内是单个单词且不是表达式时**，括号与内容之间**不加空格**
- **其他所有情况**（单个表达式、多个单词、多个表达式、空括号），括号内必须加空格

#### 函数定义规则
- 函数名与左括号之间**必须加一个空格**
- 括号内参数遵循通用括号内规则

#### 函数调用规则
- **所有函数调用，无论括号内有没有参数**，函数名与左括号之间**必须加一个空格**
- 括号内参数遵循通用括号内规则

### 命名规范（核心强制，必须严格遵守）
- **类名**：PascalCase（大驼峰），如 `StudentManager`、`ScoreCalculator`
- **函数名**：PascalCase（大驼峰），如 `ShowMenu`、`InputScores`
- **变量名**：PascalCase（大驼峰），如 `StudentCount`、`AverageScore`
- **常量名**：首字母大写，下划线分隔，如 `Max_Student_Count`、`Min_Score`
- **宏定义**：UPPER_SNAKE_CASE（全大写下划线），如 `DEBUG_MODE`、`MAX_BUFFER_SIZE`
- **全局变量**：必须添加 `G_` 前缀，如 `G_StudentCount`、`G_IsRunning`

### 注释规范（核心强制，必须严格遵守）
- **注释必须简短，只说明代码做什么，不写为什么和怎么做**
- 禁止长篇大论的注释，一行注释不超过30个字符
- 函数注释只写函数功能，不写参数和返回值说明
- 变量注释只写变量用途，不写取值范围
- 复杂逻辑只在关键位置加简短注释
- 禁止使用多行注释 `/* */`，全部使用单行注释 `//`

### 文档与日期规范
- 所有文档和开发日志中的日期必须使用**当前真实日期**，格式为：YYYY-MM-DD
- 开发日志文件名使用ISO日期格式：YYYY-MM-DD.md
- 每次开发完成后自动更新对应日期的开发日志
- 日志内容包括：今日完成功能、遇到的问题、明日计划、进度统计
