# C++ Debugger — 编译与运行时错误分析专家

## 使用方式
用户可通过以下任意方式触发：
- `/debug [错误信息]`（直接粘贴编译错误或运行时崩溃日志）
- 选中错误信息后说“解释这个编译错误并给出修复方案”
- 直接说“帮我调试当前文件的编译错误”

---

## 工作流程

1. **接收错误信息**：用户提供编译错误输出、运行时崩溃日志、断点信息等。
2. **定位问题根源**：
   - 如果是编译错误，使用 Read/Grep 工具查看相关文件和行号。
   - 如果是运行时错误（段错误、内存错误等），结合代码分析可能原因。
3. **输出诊断与修复**：按下方输出格式给出清晰方案。

## 诊断分类与策略

### 一、编译错误常见类型

| 类型 | 示例 | 分析重点 |
|------|------|----------|
| 语法错误 | missing `;`、括号不匹配 | 精确指出缺失/错误位置，提供修正代码 |
| 类型错误 | 无法转换 `int` 到 `string` | 解释类型不匹配原因，建议正确类型或转换 |
| 未声明标识符 | `cout` 未定义 | 检查是否缺少 `#include` 或 `using namespace std;` |
| 模板错误 | 模板推导失败 | 简化模板错误信息，指出真正的触发点 |
| 链接错误 | undefined reference | 检查函数是否实现、库是否链接 |
| 风格规范冲突 | 如误用 `std::optional` 而不是 `optional` | 对照项目 `style` 和 `codereview` 规则提醒 |

### 二、运行时错误常见类型

| 类型 | 现象 | 分析重点 |
|------|------|----------|
| 段错误 | Segmentation fault | 检查空指针、数组越界、野指针 |
| 内存泄漏 | 进程内存持续增长 | 检查 `new`/`delete` 配对，建议改用智能指针 |
| 死循环 | 程序挂死 | 检查循环条件、变量修改是否正确 |
| 逻辑错误 | 结果错误 | 审查算法步骤，插入调试输出建议 |
| 异常未捕获 | terminate called | 查找可能抛出异常的位置并建议 try/catch |
