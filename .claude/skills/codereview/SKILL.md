# C++ Code Review — 项目定制审查员

## 使用方式
- `/codereview [文件路径]`
- 选中代码片段后说“审查这段代码”
- 直接说“审查当前文件的代码质量和格式”

---

## 审查规则（逐条检查，缺一不可）

### 一、项目自定义格式规范（最高优先级）

#### 1. 基础格式
| # | 规则 | 检查方法 |
|---|------|----------|
| 1 | 缩进使用 **4 个空格**，绝不用 Tab | 检查缩进字符 |
| 2 | **大括号单独换行**（Allman 风格），左右大括号各占一行 | 逐行扫描 `{` `}` |
| 3 | **运算符前后必须加空格**（`+` `-` `*` `/` `=` `==` `<<` `>>` 等） | 检查运算符两侧 |
| 4 | **逗号后必须加空格** | 检查 `,` 后 |
| 5 | **引用符号 `&` 两边必须加空格**：`int & a`，`const string & str` | 检查 `&` 两侧 |
| 6 | **关键字后必须加空格**（`if` `for` `while` `switch` `return` 等） | 检查关键字后 |

#### 2. 括号与空格终极规则
| # | 规则 | 检查方法 |
|---|------|----------|
| 7 | 括号内仅单个字面量时不加空格：`(true)` `(false)` `(nullptr)` | 精确匹配 |
| 8 | 括号内其他所有情况内侧加空格：空括号 `( )`、单个变量 `( flag )`、表达式 `( a == b )`、多内容 `( int i = 0; i < n; i++ )`、字符串 `( "pause" )` | 逐处检查 |
| 9 | **函数定义**：函数名与 `(` 之间必须加一个空格，括号内遵循规则 7/8 | 检查定义 |
| 10 | **函数调用**：函数名与 `(` 之间永远加一个空格（无论有无参数），括号内遵循规则 7/8 | 检查调用 |

#### 3. 项目特定约定
| # | 规则 | 检查方法 |
|---|------|----------|
| 11 | **函数之间空一行**；函数前有注释时空两行 | 检查空行数量 |
| 12 | **命名使用大驼峰**；全局变量必须带 `G_` 前缀，如 `G_StudentList` | 正则匹配 |
| 13 | **常量首字母大写加下划线**：`Max_Student_Count` | 检查 const/constexpr |
| 14 | **注释简短，仅说明功能**，不写废话 | 阅读注释 |
| 15 | **文件开头必须有 `using namespace std;`** | 检查顶部 |
| 16 | **`main` 函数末尾必须有 `return 0;`** | 检查 main |
| 17 | **已有 `using namespace std;` 的文件禁止 `std::` 前缀** | Grep `std::` |

---

### 二、安全问题

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | 空指针/悬空指针 | 使用前判空；delete 后置 nullptr |
| 2 | 缓冲区溢出 | 数组边界检查；安全函数替代 |
| 3 | 内存泄漏 | new/delete 配对；异常路径也释放 |
| 4 | 未初始化变量 | 局部变量使用前初始化 |

---

### 三、性能问题

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | 不必要的拷贝 | 参数用 const &；循环内避免拷贝大对象 |
| 2 | 循环内重复计算 | 不变量提到循环外 |
| 3 | 低效容器操作 | 用 find 替代 [] 重复查找；emplace_back |
| 4 | 不必要的内存分配 | 循环内避免反复分配 |

---

### 四、现代 C++ 最佳实践

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | RAII | 构造获取资源，析构释放 |
| 2 | 智能指针 | unique_ptr/shared_ptr 替代裸指针 |
| 3 | const 正确性 | 不修改的成员函数加 const |
| 4 | C++17 特性 | 结构化绑定、if constexpr、optional 等 |
