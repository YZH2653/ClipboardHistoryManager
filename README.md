# 历史剪贴板管理器

一个轻量级的Windows历史剪贴板管理软件，自动记录您复制的所有内容，方便随时查看和再次使用。

## ✨ 功能特性

- **自动记录**：复制的文字和图片会自动保存到本地
- **历史查看**：按时间倒序显示所有记录，最新复制的内容排在最前面
- **搜索功能**：支持文字搜索和时间搜索（日期、时间、年月）
- **置顶功能**：重要内容可以置顶，固定在列表顶部
- **删除功能**：不需要的记录可以手动删除
- **快速复制**：点击卡片即可复制内容到剪贴板
- **过期清理**：自动清理超过保留期限的记录
- **截图去重**：10秒内相同图片只保存一次
- **便携使用**：绿色软件，无需安装，可直接放在U盘使用

## 🚀 快速开始

### 编译运行

```bash
# 编译 sqlite3.c（需要单独编译为 C 文件）
gcc -c -o sqlite3.o sqlite3.c

# 编译程序（需要 MinGW-w64）
g++ -std=c++17 -o output/ClipboardHistory.exe Main.cpp ClipboardManager.cpp Storage.cpp sqlite3.o -luser32 -lkernel32 -lgdi32 -lgdiplus -lshell32 -ladvapi32 -mwindows

# 运行程序
./output/ClipboardHistory.exe
```

### 使用方法

1. 启动程序后，它会自动监听剪贴板变化
2. 复制任何文字或图片，程序会自动记录
3. 在界面中查看、搜索、管理历史记录
4. 点击齿轮图标进入设置页面

## 🔍 搜索格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 完整日期 | `2026-05-31` | 搜索指定日期的记录 |

## 📁 项目结构

```
ClipboardHistoryManager/
├── Main.cpp                 # 主程序（入口、UI、消息循环）
├── ClipboardManager.h/cpp   # 剪贴板监听和内容捕获
├── Storage.h/cpp            # SQLite数据库存储管理
├── sqlite3.h/c              # SQLite数据库引擎
├── clips/                   # 存储目录（运行时创建）
│   ├── history.db           # SQLite数据库文件
│   └── images/              # 图片存储
├── docs/                    # 项目文档
├── versions/                # 版本开发目录
├── devlogs/                 # 软件初始开发日志
└── output/                  # 编译输出
```

## 🛠️ 技术栈

- **语言**：C++17
- **编译器**：MinGW-w64 (GCC)
- **UI框架**：Win32原生API（GDI绘制）
- **数据库**：SQLite3（v3.45.0）
- **图片处理**：GDI+

## 📋 系统要求

- Windows 7/8/10/11
- 无需额外安装任何组件

## 📦 便携版使用

1. 将 `portable/历史剪贴板管理器` 文件夹复制到U盘
2. 在其他电脑上，双击 `启动程序.bat` 或 `ClipboardHistory.exe`
3. 程序会自动创建 `clips` 文件夹存储数据
4. 数据跟随程序存储，换电脑不丢失

## ⚙️ 配置说明

- **保留天数**：默认保留最近3天的记录（可选3天/5天/7天/30天/永久）
- **最大记录数**：默认最多保存1000条记录
- **文字限制**：单条文字最多保存10000字符
