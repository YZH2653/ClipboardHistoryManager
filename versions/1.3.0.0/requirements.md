# 版本 1.3.0.0 需求文档

## 功能需求

### 1. 开机自启动

**目标**：程序可以设置为 Windows 开机自动启动

**技术方案**：
- 使用 Windows 注册表实现
- 注册表路径：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
- 键名：`ClipboardHistoryManager`
- 键值：程序 exe 的绝对路径

**功能要求**：
- 程序启动时从数据库加载自启设置
- 如果设置为自启，确保注册表中有对应项
- 如果设置为不自启，确保注册表中无对应项
- 注册表与数据库设置保持同步

### 2. 设置页面开关

**目标**：在设置页面中添加开机自启的开关按钮

**界面设计**：
- 位置：在"保存时间"和"版本信息"之间
- 左侧显示文字"开机自启"
- 右侧显示开关按钮（开/关状态）
- 开启状态：蓝色背景，显示"开"
- 关闭状态：灰色背景，显示"关"

**交互要求**：
- 点击开关切换状态
- 切换后立即保存到数据库
- 切换后立即更新注册表
- 实时反映状态变化

## 技术要求

1. 使用 Windows API 操作注册表
2. 在 Storage 中添加自启设置的保存和加载
3. 使用 SQLite settings 表存储自启状态（key: "autoStart", value: 0/1）
4. 编译时需要链接 advapi32 库（注册表操作）
