# 版本 1.5.0.0 需求文档

## 功能需求

### 1. 全选功能

**需求描述**：
在列表顶部添加全选复选框，用户可以一键选中或取消选中所有记录。

**功能要求**：
- 全选复选框显示在列表顶部
- 点击全选可选中当前显示的所有记录
- 再次点击全选可取消所有选中状态
- 全选状态与单条记录的选中状态联动

### 2. 多选功能

**需求描述**：
每条历史记录前添加复选框，支持单选或多选操作。

**功能要求**：
- 每条记录左侧显示复选框
- 点击复选框可切换选中/未选中状态
- 选中状态有明显的视觉反馈（如高亮背景）
- 支持批量操作模式切换

### 3. 批量删除功能

**需求描述**：
选中多条记录后，点击删除按钮一次性删除所有选中的记录。

**功能要求**：
- 删除按钮显示已选中记录数量
- 点击删除时弹出确认对话框
- 确认后批量删除所有选中记录
- 删除后自动刷新列表显示
- 删除操作不可撤销，需谨慎处理

### 4. 垃圾桶按钮

**需求描述**：
在标题栏添加垃圾桶按钮，点击可进入/退出选择模式。

**功能要求**：
- 垃圾桶按钮显示在标题栏右侧（设置按钮左边）
- 点击垃圾桶按钮进入选择模式
- 再次点击垃圾桶按钮退出选择模式
- 选择模式下垃圾桶按钮显示红色背景
- 退出选择模式时清空所有选中状态

### 5. 界面交互

**需求描述**：
优化界面交互，提供清晰的操作反馈。

**功能要求**：
- 选中记录有明显的视觉标识
- 显示已选中记录的数量
- 批量删除按钮在未选中时禁用
- 操作完成后显示成功/失败提示

## 技术要求

### 1. 数据结构

- 使用 `std::vector<int>` 存储选中记录的ID
- 使用 `std::map<int, bool>` 存储每条记录的选中状态
- 全选状态使用布尔变量控制

### 2. UI绘制

- 在标题栏绘制垃圾桶按钮（设置按钮左边）
- 在列表顶部绘制全选复选框
- 每条记录前绘制单个复选框
- 选中状态使用不同背景色区分
- 绘制批量删除按钮和选中计数

### 3. 消息处理

- 处理垃圾桶按钮的点击事件（进入/退出选择模式）
- 处理全选复选框的点击事件
- 处理单条记录复选框的点击事件
- 处理批量删除按钮的点击事件
- 处理确认对话框的响应

### 4. 数据库操作

- 实现批量删除SQL语句
- 使用事务保证数据一致性
- 删除后更新内存中的记录列表
