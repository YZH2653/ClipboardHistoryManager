# 版本开发指引

本文件夹记录各版本的开发过程、需求、进度和日志。

## 目录结构

```
versions/
├── README.md                    # 本文件：版本开发指引
├── X.X.X.X/                     # 各版本文件夹
│   ├── README.md                # 版本概述
│   ├── changelog.md             # 版本更新日志
│   ├── requirements.md          # 本版本需求
│   ├── execution-steps.md       # 执行步骤
│   └── devlogs/                 # 开发日志（按日期）
│       └── YYYY-MM-DD.md
```

## 版本号规范

采用 `X.X.X.X` 格式：
- 第1位：主版本号（重大重构或不兼容变更）
- 第2位：次版本号（新功能添加）
- 第3位：修订号（Bug修复）
- 第4位：构建号（小改动）

## 开发流程

1. **创建分支**：从 main 分支创建版本分支 `git checkout -b X.X.X.X`
2. **编写文档**：在 `versions/X.X.X.X/` 下创建需求、执行步骤等文档
3. **逐步开发**：按执行步骤逐步实现功能，每完成一步提交一次
4. **测试验证**：功能开发完成后进行全面测试
5. **合并主干**：测试无误后合并到 main 分支
