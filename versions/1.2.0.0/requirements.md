# 版本 1.2.0.0 需求文档

## 功能需求

### 1. 修复历史记录丢失

**问题**：JSON 文件存储存在路径编码问题，导致历史记录丢失

**解决方案**：
- 使用 SQLite 数据库替代 JSON 文件存储
- 通过 GetModuleFileNameA 获取 exe 绝对路径
- 使用事务处理确保数据一致性

### 2. 修复截图重复保存

**问题**：系统截图工具（Win+Shift+S）会多次触发剪贴板更新，每次序列号都不同

**解决方案**：
- 使用图片哈希比较替代序列号检查
- 取 DIB 数据前 1024 字节计算快速哈希
- 哈希相同 + 间隔<10秒 → 跳过保存

### 3. 修复图片保存失败

**问题**：G_ClipManager.SetRootDir 未调用，导致 m_rootDir 为空

**解决方案**：
- 在 Main.cpp 的 main() 函数中添加 SetRootDir 调用
- 确保图片保存路径正确

### 4. 移除调试输出

**问题**：程序运行时弹出 cmd 窗口显示调试信息

**解决方案**：
- 移除所有 wcout 调试输出
- 编译参数添加 -mwindows

## 技术要求

1. 集成 SQLite3 数据库引擎
2. 重写 Storage 模块使用 SQLite
3. 实现图片哈希去重逻辑
4. 保持原有功能完整
